Registers:

Letter Name Size Function
A Accumulator 16 bits Math Operations, Data Transfer
B Buffer 16 bits Store Temporary Data
X X - Index 16 bits Store Address Offsets
Y Y - Index 16 bits Store Address Offsets
I Instruction Pointer 16 bits Store Next Opcode Address
S Stack Pointer 12 bits Store Next Stack Entry
P Processor Status 8 bits Store Runtime Flags
  • Accumulator is dividen on 2 8-bit parts: Data (D) and High (H), though only Data is available for direct access.
  • Stack Pointer will point at addresses $1000 - $1FFF.
  • Stack Limiter (V) will set the lower bounds of stack.

Processor Status:

Bits: NV--FIZC

Letter Name Function
N Negative Is Bus Data Negative
V Overflow Overflow after Math
F Float Mode ALU Number Type
I Interrupt Disable IRQs
Z Zero Is Bus Data equal to Zero
C Carry Carry after Operation

Addressing Modes:

Notation Name Size Description
IMP Implied - Opcode does one specific action

B Buffer - Operation with Buffer

DIM Immediate Word 2 bytes Operation with Next Word
REL Relative 1 byte Operation with Relative Address
DIR Direct 2 bytes Operation with Data at Next Absolute Address
DRX Direct + X 2 bytes Operation with Data at Next Absolute Address + X
DRY Direct + Y 2 bytes Operation with Data at Next Absolute Address + Y
ZPG Zeropage 1 byte Operation with Data at Next Zeropage Address

ZPY Zeropage + Y 1 byte Operation with Data at Next Zeropage Address + Y

  • Zeropage is located at addresses $0000 - $00FF.
  • Accumulator and Buffer modes are technically the same as Implied mode, but marked for user's convenience. They usually can be written as Implied mode.

Vectors:

To specify where the CPU should start to execute the code, ROM should contain vectors that point to interrupt entries:

Address Name Interrupt Description
$FFFA - $FFFB IRQ Yes Interrupt that can be triggered by CPU pin or by software. IRQ Flag will not forbid hardware interrupts.
$FFFC - $FFFD RST No Program entry for the CPU. Instruction Pointer will be set to RST vector after reseting the CPU.
$FFFE - $FFFF NMI Yes Non-maskable Interrupt is triggered by hardware pin. IRQ Flag does not affect NMI triggering.

Subtraction:

ALU performs subtraction by using addition:
 SBC = A + (not B) + C

This leads to the next results:

Flags after Subtraction (a - b):

Numbers N Z C
a > b ? 0 1
a = b 0 1 1
a < b ? 0 0

Branches as Logical Conditions:

Opcode Sign Logical Condition
BEQ = Equal
BNE != Not Equal
BCC < Less
BCS >= Greater or Equal

Console

Processor:

This console uses X65 CPU (described earlier). As console needs to communicate to different devices at once, different memory locations are mapped to different devices.

Address Device
$0000 - $3FFF RAM
$4000 - $5FFF Registers
$6000 - $7FFF SRAM
$8000 - $FFFF ROM
  • Register space is not fully filled.
  • SRAM is Save RAM; It supports 8 banks.
  • ROM is dividen on 8 banks, each containing 4096 bytes.
  • There can be up to 256 ROM banks.
  • First ROM bank is the default bank.

Due to small amount of memory accessible by CPU, external banks are provided.

Bank Registers:

Address Name
$4010 Bank 0 ($8000 - $8FFF)
$4011 Bank 1 ($9000 - $9FFF)
$4012 Bank 2 ($A000 - $AFFF)
$4013 Bank 3 ($B000 - $BFFF)
$4014 Bank 4 ($C000 - $CFFF)
$4015 Bank 5 ($D000 - $DFFF)
$4016 Bank 6 ($E000 - $EFFF)
$4017 Bank 7 ($F000 - $FFFF)
$4018 Bank S (SRAM Bank)

Control:

For input, console uses 2 joysticks.

Address Name

$5002 - $5003 Joystick 2

Fetched data from registers comes in specific form:

Graphics:

Console outputs graphics using GPU.

  • To create graphics, GPU uses 8x8 tiles, stored in a seperate CHR ROM with a total amount of 2048 tiles.
  • Each tile has bit depth of 4bpp (bits per pixel), that fetches the colors from one of 16 availabe palettes.
  • The Global Palette consists of 256 colors (16 colors per each palette), which has RGBA 4444 format.
  • GPU output image is 40x30 tiles, or 320x240 pixels.
  • The background color of image is the first color in the Global Palette.
  • Output consists of 2 layers, each having 80x60 tiles.
  • Each layer can be individually scrolled.
  • GPU supports 128 Sprites - independant tiles on the screen.
  • Sprites can be drawn below, between or above background layers.

CPU can control GPU by writing to its registers, that are listed below.

GPU registers:

Address Name Function

$4002 GPU Control Register Register for general control
$4003 GPU Room Register Register for selecting base rooms for scrolling
$4004 - $4005 GPU VRAM Address Set the value of Address Pointer
$4006 - $4007 GPU SPR Address Address, where the sprite data will be fetched.
$4008 - $4009 GPU Layer 0 X-Scroll X - Scrolling for Layer 0
$400A - $400B GPU Layer 0 Y-Scroll Y - Scrolling for Layer 0
$400C - $400D GPU Layer 1 X-Scroll X - Scrolling for Layer 1
$400E - $400F GPU Layer 1 Y-Scroll Y - Scrolling for Layer 1

N - Trigger NMI every frame

  • 00 - Move Right
  • 01 - Move Down
  • 10 - Move Left
  • 11 - Move Up

Sprites:

Sprite Data will be grabbed from the CPU RAM during the beginning of rendering the frame.

It will be fetched from the Sprite Address set in the GPU. 768 bytes will be fetched.

Offset Entry Size Description
$000 - $0FF 2 bytes X Position. Uses Signed Integer
$100 - $1FF 2 bytes Y Position. Uses Signed Integer
$200 - $2FF 2 bytes Attribute Data

Attribute Data:
PPPPLLTT TTTTTTTT (Big Endian)

P - Palette ID
T - Tile ID
L - Layer
  • 00 - Under Layer 0
  • 01 - Between Layers
  • 1x - Above Layer 1

Audio:

APU is a seperate processor that produces sound.

  • APU has 8 channels that playback samples.
  • There can be up to 128 samples, that are made of 256 8-bit samples.
  • Stereo audio is also supported.

As GPU, APU is controlled by registers.

Cartridge Mapping:

As cartridge data is made up of 3 ROMs, it is packed in a single file in a certain way.

At the beginning of file 16-byte Header is located.

Header Structure:

Offset Values Description
$00 - $03 "x65\0" Signature
$04 - $05 0 - 256 PRG ROM Bank count (Bank size = 4096)
$06 0 - 128 DSD ROM Bank count (Bank size = 256)
$07 - $0F - Unused

It is followed by the specified amount of PRG and DSD banks. The remaining data will be treated as CHR ROM.
